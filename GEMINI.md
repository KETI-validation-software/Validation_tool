# Project Mandates

## Workflow Requirements
- **Mandatory Workflow Steps (Strict Order):**
    1.  **Confirmation & Understanding:** Begin by summarizing the user's request to confirm you understand it correctly.
    2.  **Clarifying Questions:** Ask any necessary follow-up questions to resolve ambiguities or gather missing information.
    3.  **Implementation Strategy:** Present a detailed plan and code changes before executing them.
    4.  **Wait for Approval:** Do NOT modify files until the user has explicitly confirmed the plan.
    5.  **Execution:** Only after confirmation, proceed with implementing and validating the changes.

## Engineering Standards
- All code changes must be idiomatically consistent with the existing codebase.
- Verification (testing or manual proof) is required for every implemented change.
