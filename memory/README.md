# Memory Workspace Guide

This folder is intentionally gitignored for local project management notes.

## Purpose
- Keep project understanding up to date.
- Track implemented features and known gaps.
- Maintain feedback backlog and execution priorities.
- Record key decisions to prevent repeated debates.

## Recommended update flow
1. Update project snapshot when scope changes.
2. Add or revise feature status after implementation.
3. Record architecture/design changes.
4. Add feedback items with priority and owner.
5. Append decision log for important trade-offs.

## Archive policy
- Older or superseded notes should move to `memory/archive/`.
- Keep raw history only when it adds context not preserved by a canonical file.
- Prefer consolidating related dated notes into one durable summary document.

## Rules
- Keep entries short and practical.
- Prefer factual updates over long narrative.
- Add date for each meaningful update.
- Treat `worklog.md` as legacy until it is intentionally normalized or rebuilt.
